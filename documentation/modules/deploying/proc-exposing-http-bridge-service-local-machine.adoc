= Exposing the HTTP Bridge service to your local machine

[role="_abstract"]
Use port forwarding to access the HTTP Bridge locally at \http://localhost:8080.

Port forwarding is suitable for development and testing only.

.Procedure

. List the pods in your Kubernetes cluster:
+
[source,shell,subs=attributes+]
----
kubectl get pods -o name
----
+
Identify the pod associated with your HTTP Bridge deployment, for example:
+
----
pod/my-bridge-bridge-<pod_id>
----

. Forward local port `8080` to port `8080` in the HTTP Bridge pod:
+
[source,shell,subs=attributes+]
----
kubectl port-forward pod/my-bridge-bridge-<pod_id> 8080:8080
----
+
If port `8080` is already in use on your local machine, use an alternative local port, for example:
+
----
kubectl port-forward pod/my-bridge-bridge-<pod_id> 8008:8080
----

Requests sent to \http://localhost:8080 are forwarded to the HTTP Bridge pod.
